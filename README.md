# АС-ФИД — Корпоративный сайт

Корпоративный сайт компании ООО «АС-ФИД» — российского производителя экструдированных кормов для аквакультуры.

## Структура проекта

```
АС-ФИД САЙТ/
├── frontend/           # Основной сайт
│   ├── index.html      # Главная страница (SPA)
│   ├── css/
│   │   └── styles.css  # Стили
│   ├── js/
│   │   └── main.js     # JavaScript
│   ├── images/         # Изображения
│   ├── robots.txt      # Для поисковых систем
│   └── sitemap.xml     # Карта сайта
│
├── admin/              # Админ-панель для отчётности
│   └── index.html      # Панель управления заявками
│
├── backend/            # Сервер
│   ├── server.js       # Node.js сервер
│   ├── package.json    # Зависимости
│   ├── .env.example    # Пример конфигурации
│   └── data/           # Хранение заявок (создаётся автоматически)
│
└── README.md           # Этот файл
```

## Быстрый старт

### 1. Установка зависимостей

```bash
cd backend
npm install
```

### 2. Настройка конфигурации

```bash
cp .env.example .env
```

Отредактируйте `.env` файл:

```env
# Обязательно: токен для админ-панели
ADMIN_TOKEN=ваш-секретный-токен

# Опционально: настройки email уведомлений
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=ваш-email@yandex.ru
SMTP_PASS=пароль-приложения
NOTIFICATION_EMAIL=info@as-feed.ru
```

### 3. Запуск сервера

```bash
npm start
```

Сервер запустится на `http://localhost:3000`

## Доступ

- **Основной сайт**: `http://localhost:3000`
- **Админ-панель**: `http://localhost:3000/admin`

Для входа в админ-панель используйте токен из `.env` файла.

## Функциональность

### Основной сайт

- **Главная страница** — презентация компании и УТП
- **О компании** — информация о АС-ФИД
- **Каталог кормов** — продукция с фильтрацией по видам рыб:
  - Форель
  - Осётр
  - Карп
  - Сом
  - Тилапия
- **Партнёры** — блок для партнёров и инвесторов
- **Контакты** — форма обратной связи

### Админ-панель

- Статистика заявок (всего, сегодня, за неделю)
- Список всех заявок с фильтрацией по статусу
- Детальный просмотр заявки
- Изменение статуса заявки (новая, в работе, завершена, отклонена)

## API Endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/submit-form` | Отправка формы заявки |
| GET | `/api/leads` | Получение списка заявок (требует токен) |
| PATCH | `/api/leads/:id` | Обновление статуса заявки (требует токен) |
| GET | `/api/stats` | Статистика заявок (требует токен) |
| GET | `/api/health` | Проверка состояния сервера |

## Деплой на продакшн

### Вариант 1: VPS/Dedicated сервер

1. Установите Node.js 18+
2. Склонируйте проект на сервер
3. Настройте `.env`
4. Используйте PM2 для запуска:

```bash
npm install -g pm2
cd backend
pm2 start server.js --name "as-feed"
pm2 save
```

5. Настройте Nginx как reverse proxy
6. Получите SSL сертификат (Let's Encrypt)

### Вариант 2: Статический хостинг + Serverless

Frontend можно развернуть на:
- Vercel
- Netlify
- GitHub Pages

Backend развернуть на:
- Vercel Serverless Functions
- Railway
- Render

## Замена плейсхолдеров

После получения материалов от заказчика замените:

1. **Логотип** — SVG в `index.html` (поиск по `logo-icon`)
2. **Контакты**:
   - Телефон: поиск по `+7 (XXX)`
   - Email: поиск по `info@as-feed.ru`
   - Адрес: поиск по `уточняется`
3. **Партнёры** — блоки в секции `#page-partners`

## Технологии

- **Backend**: Node.js, Express
- **Хранение данных**: JSON файлы (легко мигрировать на БД)
